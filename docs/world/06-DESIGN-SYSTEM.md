# 06 — Math Learning World Design System

## Purpose

The Math Learning World Design System (MLWDS) keeps world design, learning, gameplay, art, UI, animation, and audio aligned as the project grows.

It is a decision system, not merely a component library.

## Shared design grammar

Every production element should express at least one of these verbs:

- notice;
- compare;
- arrange;
- build;
- test;
- predict;
- repair;
- explain;
- connect;
- contribute.

## Cross-discipline contract

### World design
Creates meaningful places, routes, systems, and consequences.

### Learning design
Defines the mathematical relationship, misconception, evidence, and transfer path.

### Gameplay
Turns the relationship into player action and feedback.

### Art
Makes hierarchy, state, scale, and emotional purpose visible.

### UI
Supports decisions without replacing observation of the world.

### Animation
Communicates cause, state transition, weight, rhythm, and response.

### Audio
Reinforces material, distance, success, tension, pattern, and environmental identity.

## Feature acceptance canvas

A feature is not ready for implementation until it states:

1. Player need or curiosity.
2. Primary verb.
3. World consequence.
4. Learning meaning, if any.
5. Feedback loop.
6. Visual and audio hierarchy.
7. Accessibility considerations.
8. Evidence of completion or mastery.
9. Relationship to an existing region or system.
10. Runtime and production constraints.

## UI principles

- One primary action per moment when possible.
- Prefer direct manipulation over menus.
- Keep language simple and translatable.
- Use symbols only when their meaning is learnable and consistent.
- Do not cover the phenomenon the player is meant to observe.
- Show consequences in the world before summarizing them in panels.

## Interaction principles

- Nearby context should reduce unnecessary tool switching.
- The system may assist selection, but the player must retain understandable control.
- Repeated actions should become smoother without hiding causality.
- Interactions must communicate valid targets, current state, and reversibility.

## Content growth rule

New regions and systems should reuse the shared grammar while introducing a distinct emotional promise and mathematical perspective. Reuse should create coherence, not sameness.

## Governance

Changes to foundational principles require explicit review. Production details may evolve through evidence from runtime slices, player observation, and learning validation.

When doctrine conflicts with verified player evidence, the conflict must be documented and the doctrine revised deliberately rather than ignored silently.
