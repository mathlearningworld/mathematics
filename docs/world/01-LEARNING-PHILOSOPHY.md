# 01 — Learning Philosophy

## Purpose

This document defines how Math Learning World turns play into durable mathematical understanding.

## Core sequence

```text
Experience → Notice → Wonder → Try → Compare → Explain → Transfer
```

Instruction may support this sequence, but should not routinely replace it.

## Governing principles

### Wonder before knowledge
Create a compelling phenomenon, place, problem, or pattern before presenting terminology.

### Curiosity before explanation
Give the player enough information to begin forming questions. Explanations should resolve meaningful curiosity rather than interrupt it.

### Discovery before instruction
Whenever safe and practical, allow the player to test relationships through action before formal teaching.

### Build before memorize
Construction, arrangement, measurement, prediction, and repair should establish meaning before symbolic procedures are emphasized.

### Understand before formula
A formula should compress an already meaningful relationship. It must not be the player's first encounter with that relationship.

### Apply before examination
Players should use ideas in authentic situations before being asked to demonstrate mastery under assessment conditions.

## Mastery model

Mastery is not a single correct answer. Evidence should combine:

- accuracy;
- strategy selection;
- explanation or representation;
- transfer to a changed context;
- recovery after feedback;
- retention over time.

Difficulty should adapt to the player's actual dependency gaps, not merely their school grade.

## Remediation doctrine

Remediation must return to meaning, not simply repeat the same prompt.

A remediation loop should vary at least one of:

- representation;
- context;
- manipulable object;
- scale;
- feedback mode;
- prerequisite concept.

Repeated difficulty is diagnostic evidence. It should trigger a gentler path, prerequisite repair, or human support—not stigma.

## Feedback doctrine

Good feedback tells the player what the world did and why.

Prefer:

- visible consequences;
- contrasts between attempts;
- partial success;
- specific clues;
- opportunities to revise.

Avoid:

- unexplained red marks;
- public comparison;
- excessive interruption;
- rewards that overshadow the idea.

## Motivation doctrine

The primary motivations are:

- curiosity;
- competence;
- creation;
- belonging;
- contribution;
- access to new possibilities.

Extrinsic rewards may support these motivations, but must not become the main reason to engage.

## Parent and teacher role

Adults should see meaningful evidence: what the learner understands, where dependencies are weak, what strategies they use, and what kind of support is likely to help.

The system should not reduce a child to one score.

## Learning design review questions

Before accepting a learning activity, verify:

1. What meaningful experience creates the need for this idea?
2. What can the player manipulate, compare, or predict?
3. What misconception can the activity reveal?
4. How does feedback help revision?
5. Where will the player transfer the idea next?
6. What evidence proves understanding beyond recall?
